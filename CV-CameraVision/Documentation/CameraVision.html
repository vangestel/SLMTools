CameraVision.vi

This vi is the main component of the CameraVision software.

The software is made for the acquisition, processing and storage of images. Specifically using one or two CMOS cameras and a SiliconSoftware framegrabber. The cameras can be any type compatible with the framegrabber, though the software is only tested with PhotonFocus CMOS-cameras of the MV-D1312-240-CL model. In this case a separate software called 'PFRemote', provided by PhotonFocus, is necessary for changing camera parameters.

The main program controls are on the left side of the window. User-configurable settings are grouped per category on the lower left panel. All outputs are displayed at the right side of the window. Different display configurations can be chosen using the tabs.

 Start acquisition

Pressing this button initializes the cameras and starts acquiring and processing images.

Depending on the chosen 'Acquisition Mode', the acquisition starts immediately or a the time specified in the 'Start Time' field on the 'Timing' tab.

Pause acquisition

Pause or unpause (restart) the acquisition and processing of images.

Restarting after a pause does not re-initialize the cameras. No changes can be made to the program settings during a pause.

Attention: During a pause, no new images are buffered and thus information is lost.

Stop acquisition

Pressing this button stops the acquisition and processing of images and releases the camera resources.

If the button is pressed while a custom Start timer is active, the timer is de-activated and the camera resources are released.

Settings type

The source of the program settings. No action is taken untill the 'Load settings' button is pressed.

Values:

Custom - The default setting. Values can be freely changed by the user.

Default - The settings are reset to the programmed default values.

Last used - The settings are reset to the values used during the last acquisition. Even when the software has since been restarted.

Saved - The settings are reset to the values of a previously saved configuration file. The configuration file has to be selected by the user through a dialog box, presented when the 'Load settings' button is pressed.

The default value is Custom.

Load settings

Pressing this button loads or resets the settings parameters, according to the value selected in the 'Settings source' field.

Save settings

Save the current settings parameters in a separate settings file. The file format is .ini.

Note: When starting an acquisition, the current settings are already automatically saved in a settings file in the 'Storage directory' with the 'Base file name' as name.

Import images

This function allows the re-processing of previously acquired images. Saved images are selected using a pop-up browser window. The re-processing can be started, paused or stopped like a normal acquisition. To exit the re-processing mode, click 'Unload'.

Warning: The re-processed images do not inherit the time-stamp from the original images.

Note: Re-loading images from memory can be time-consuming. Consider using a low sample rate.

Note2: Two cameras can be simulated using the default naming pattern for two cameras and selecting '2' in the 'Number of cameras' field in the configuration settings.

Display camera

Select which camera image/information to display.

This option is only active when 2 cameras are used.

The default value is Camera 1.

Display image

Select which image type to display.

Values:

Original - The originally captured image without any changes or processing.

Corrected - The image after background, hot-pixel correction, symmetry correction, spatial correction and/or smoothing.

Binary - The image after binarization (black-white) using the specified 'Binarization treshold'.

The default value is the Original image.

Browse batch

Browse through all the images stored in the current image batch.

This option is only available when the 'Single batch' or 'Continuous batch' modes are selected on the 'Acquisition' tab.

The image at position 0 is always the last acquired image.

Configuration params.

Settings that determine the configuration of the camera hardware.

    		Camera config file

    The path to the configuration file for the Photonfocus cameras.

    Only mcf-type files are accepted.

    It is recommended to refer to the configuration file included with this software or a copy thereof. The file is located in the 'Support files' folder.

    		Number of cameras

    The number of cameras to use. The cameras have to be connected to the aqcuisition card (framegrabber) at port 0.

    The default value is 1.

    Accepted values are 1 and 2.

    		Camera 1 port

    The letter of the port on the data-acquisition card to which the first camera (Camera 1) is connected.

    The default value is A.

    Accepted values are A and B.

    When two cameras are used, the second letter is assigned automatically.

    		ROI width [pixels]

    The number of pixels to read in the x-direction.

    This defines the width of the image that is processed and stored.

    Attention: The value has to be the same as the one specified in the 'PFRemote' software on the 'Window' tab (parameter W). Except for values below 480, the 'PFRemote' value should remain 480.

    The default value is 1248 pixels.

    Accepted values are 96-1248, in steps of 96 pixels.

    		ROI height [pixels]

    The number of pixels to read in the y-direction.

    This defines the height of the image that is processed and stored.

    Attention: the value has to be the same as the one specified in the 'PFRemote' software on the 'Window' tab (parameter H).

    The default value is 1082 pixels.

    Accepted values are 1-1082, in steps of 1 pixel.

    		Reset ROI

    Reset the ROI width and ROI height to their respective maximum values.

    		Width scale [mm/pixel]

    The scaling used for, by example, the calculation of the real-life size of the melt pool.

    A calibration image is needed for the determination of the scale. If the calibration is not known, a value of '1' is recommended. In this case, the size would equal the number of pixels.

    		Height scale [mm/pixel]

    The scaling used for, by example, the calculation of the real-life size of the melt pool.

    A calibration image is needed for the determination of the scale. If the calibration is not known, a value of '1' is recommended. In this case, the size would equal the number of pixels.

    		Center Y cam. 1

    The pixel position of the center of the Region Of Interest (ROI) field of camera 1.

    This is the y-coordinate of the pixel that represents the center of region to be imaged.

    The y-coordinate is relative to the top-left corner of the full (1248x1082 pixels) image.

    The default is 541 (center of the image sensor).

    The center is automatically shifted if the ROI overlaps with the image sensor borders.

    		Cam. 1 center (X,Y)

    The pixel position of the center of the Region Of Interest (ROI) field of camera 1.

    This is the x-coordinate of the pixel that represents the center of region to be imaged.

    The x-coordinate is relative to the top-left corner of the full (1248x1082 pixels) image.

    The default is 624 (center of the image sensor).

    The center is automatically shifted if the ROI overlaps with the image sensor borders.

    		Center Y cam. 2

    The pixel position of the center of the Region Of Interest (ROI) field of camera 2.

    This is the y-coordinate of the pixel that represents the center of region to be imaged.

    The y-coordinate is relative to the top-left corner of the full (1248x1082 pixels) image.

    The default is 541 (center of the image sensor).

    The center is automatically shifted if the ROI overlaps with the image sensor borders.

    		Cam. 2 center (X,Y)

    The pixel position of the center of the Region Of Interest (ROI) field of camera 2.

    This is the x-coordinate of the pixel that represents the center of region to be imaged.

    The x-coordinate is relative to the top-left corner of the full (1248x1082 pixels) image.

    The default is 624 (center of the image sensor).

    The center is automatically shifted if the ROI overlaps with the image sensor borders.

Timing params.

Settings that determine the timing and time constraints of the image acquisition.

    		Acquisition start

    Immediately start or post-phone the acquisition.

    When 'Immediate' is selected, the acquisition will start immediately after the 'Start' button is pressed and the cameras are initialized.

    When 'Post-phone' is selected, pressing the 'Start' button will initialize the cameras, but acquisition will be delayed untill the time specified in the 'Start time' field.

    		Duration MM

    The maximum duration of the image acquisition.

    The time is counted from the actual start of the acquisition (the 'Start time' value in case of a post-phoned start).

    When the time has passed, any acquisition still busy will be stopped as if the 'Stop' button was pressed.

    The default value is 1 hour.

    		Max. duration [HH:MM:SS]

    The maximum duration of the image acquisition.

    The time is counted from the actual start of the acquisition (the 'Start time' value in case of a post-phoned start).

    When the time has passed, any acquisition still busy will be stopped as if the 'Stop' button was pressed.

    The default value is 1 hour.

    		Duration SS

    The maximum duration of the image acquisition.

    The time is counted from the actual start of the acquisition (the 'Start time' value in case of a post-phoned start).

    When the time has passed, any acquisition still busy will be stopped as if the 'Stop' button was pressed.

    The default value is 1 hour.

    		Start time

    The time at which to start the acquisition of images.

    This field is only used when the 'Acquisition start' field is set to 'Post-phone'.

    If the start time is before the time at which the 'Start' button is pressed, the acquisition will start immediately.

    		Max. frame rate [frames/s]

    The maximum allowed image acquisition rate.

    If the camera captures images at a faster rate, this setting will force the acquisition to the specified value.

    The default value is 5 images/second.

    Accepted values are 0.0001-20 000 images/second.

    		Exposure time cam. 1 [ms]

    Set the exposure time for Camera 1. A longer exposure time allows light to stimulate the sensor for a longer time, resulting in a brighter image. A longer exposure can cause saturation and motion blurring.

    The default value is 1 milisecond.

    Accepted values are everything larger than 0.01 miliseconds.

    		Exposure time cam. 2 [ms]

    Set the exposure time for Camera 1. A longer exposure time allows light to stimulate the sensor for a longer time, resulting in a brighter image. A longer exposure can cause saturation and motion blurring.

    The default value is 1 milisecond.

    Accepted values are everything larger than 0.01 miliseconds.

Image refresh

Refresh the display after changing a selector value or toggle the continuous update of the display during an acquisition.

During an acquisition, this button enables or disables the continuous update of the displays.

Disabling the updating can speed up the program, while images are still acquired, processed and stored (in constrast to the 'Pause' button).

Outside an acquisition this button refreshes the current display, for example after one of the selector values is changed (Camera selector, Type selector).

Storage params.

Settings that determine where and which images and data are stored.

    		Base file name

    The first part of the name given to all files that are stored during this image acquisition. It serves as an identification for this particular acquisition.

    If a file with the same base name is found, a warning message is presented before a new acquisition is started.

    The default value is 'Test'.

    		Storage directory

    The default directory for all files that are stored during this image acquisition.

    The default value is the 'C:\CameraVision\Results'.

    		Save what?

    Select which images or data to store.

    The 'Background correction' or 'Hot-pixel correction' setting on the 'Processing' tab have to be checked in order to be able to store the 'Corrected' image.

    The 'Image characteristics' or 'Melt-pool characteristics' setting on the 'Processing' tab have to be checked in order to be able to store the 'Characteristics' data.

    The 'Temperaturefield' setting on the 'Processing' tab has to be checked in order to be able to store the 'Temperature field' data.

    The default is all unchecked.

      		Original image

      The original image.

      		Corrected image

      The image after all the correction steps.

      In order to save a Corrected image at least one of the options of the 'Correction' settings on the 'Processing' tab has to be enabled.

      		Binary image

      A binarized (black/white) version of the image, used for example for melt pool detection.

      In order to save a Binary image either the 'Melt-pool characteristics' or 'Melt-pool spatial characteristics' settings have to be enabled in the 'Analysis' section of the 'Processing' tab.

      		Temperature field (.tsv)

      A 2D numeric array of the tempearuture [°C] at each pixel.

      In order to save the temperature field, the number of cameras has to be equal to 2 and 'Temperature field' and '2nd image correlation' have to be enabled on the 'Processing' tab.

      		Characteristics (.tdms)

      Information on the general image and/or the melt pool, along with the image timestamps.

      The 'Image characteristics' or 'Melt-pool characteristics' setting on the 'Processing' tab have to be checked in order to be able to store the 'Characteristics' data.

      		Signal map (.tsv)

      A 2D colored map, correlating the position of the laser beam with the value of one of the melt pool characteristics.

      In order to save a characteristics map, 'Enable mapping' and either 'Image characteristics' or 'Melt-pool characteristics' have to be enabled on the 'Processing' tab.

      		Temperature values (.tdms)

      Statistical data related to the calculated temperature field.

      In order to save the temperature value statistics, the number of cameras has to be equal to 2 and 'Temperature field', 'Temperature value' and '2nd image correlation' have to be enabled on the 'Processing' tab.

      		Characteristics - spatial (.tdms)

      Information about the position and orientation of the melt pool.

      In order to save the spatial Melt-pool characteristics the corresponding setting has to be enabled in the 'Analysis' section of the 'Processing' tab.

    		Image file type

    The image file type in which to save the images.

    Values are JPG, BMP and PNG.

    The default value is BMP.

    BMP storess the files with lossless compression, preserving all the information. JPG and PNG store the files with lossy compression, meaning some information is lost.

    Internally, all images are stored uncompressed during the processing.

Acquisition params.

Settings that determine the way in which images are acquired.

    		Acquisition mode

    The aqcuisition mode determines the way in which images are buffered in the processing module during acquisition.

    Values:

    Continuous - In continuous mode a single image is stored in the proces module. This is the fastest mode with the lowest memory consumption. In this mode the time selector is disables, as no older images are retained in the processing module. Acquisition continuous untill the 'Stop' button is pressed or the 'Max. duration' time has elapsed.

    Single batch - A number of images determined by the 'Batch size' parameter is acquired. Acquisition is stopped automatically. The process module buffer in which the images are stored is independent from the buffer set in the 'Configuration' tab. All images in the batch can be viewed during and after acquisition using the 'Time selector' slider among the main controls.

    Continuous batch - Works the same as the 'Single batch' mode, however, acquisition is not stopped after the first batch. The x last images, with x the number in 'Batch size' can be viewed during and after acquisition using the 'Time selector' slider among the main controls.

    The default value is 'Continuous'.

    		Batch size

    The number of images buffered in the process module.

    Images that are buffered can be viewed during and after the acquisition using the 'Time selector' slider among the main controls.

    No images are buffered in the processing module when the 'Acquisition mode' is set to 'Continuous'.

    The default value is 100 images.

    Recommended values are 10-100.

    Note: The larger the batch size, the more memory is used. Using image buffering also lowers the program performance.

    		Image buffer size

    The number of images that can be temporarily stored in case of a computer or software slowdown.

    The current state of the buffer is displayed on the 'Runtime info' tab.

    If the buffer gets full during acquisition, try increasing the 'Image buffer size' or reducing the maximum sample rate on the 'Acquisition' tab.

    		Trigger mode

    The trigger mode determines how the camera is signalled to acquire an image.

    Values:

    Free running - In this mode the camera tries to approximate the chosen maximum sample rate. When two cameras are in use, they act independently.

    Hardware timed - In this mode the image acquisition harware card (framegrabber) sents trigger pulses to the camera at the chosen (max.) sample rate. When two cameras are in use they are synchronized, with the start of the exposure for each camera as the synchronization point.

    Software triggered - In this mode the cameras acquire in image when a special software command is received. This command can come from an external module (for synchronization with a laser beam, for example). When two cameras are in use they are synchronized together.

    The default mode is hardware timed.

Processing

Correction params.

    		Background correction

    This setting enables the background noise compensation during the processing of the image. The result is stored in the 'Corrected' image type.

    A reference to a black image has to be provided for each camera using he fields 'Black image camera 1' and 'Black image camera 2'. These black images can be acquired by closing the diaphragm of the cameras.

    This setting has to be checked in order to be able to store the 'Corrected' image.

    		Hot-pixel correction

    Currently disabled functionality.

    		Black image cam. 1

    Reference to a previously captured black image for Camera 1.

    This reference is necessary when 'Background correction' or 'Hot-pixel correction' are checked. A black image can be acquired by closing the diaphragm of the camera.

    		Black image cam. 2

    Reference to a previously captured black image for Camera 2.

    This reference is necessary when 'Background correction' or 'Hot-pixel correction' are checked. A black image can be acquired by closing the diaphragm of the camera.

    		Y-shift

    The shift of the 2nd camera image along the y-axis, in pixels.

    The positive y-axis is towards the bottom of the image.

    This setting has no effect unless 'Fixed shift' is selected as the correlation algorithm.

    		X-shift

    The shift of the 2nd camera image along the x-axis, in pixels.

    The positive x-axis is towards the right of the image.

    This setting has no effect unless 'Fixed shift' is selected as the correlation algorithm.

    		Correlation algorithm

    The correlation algorithm used for the superposition of images from two separate cameras.

    Values:

    Fixed shift - The second image is shifted by a user-specified amount (X-shift, Y-shift). The positive x-axis is to the right, the positive y-axis is to the bottom.

    Cross-correlation - The shift is determined by calculating the position of the maximum of the normalized cross-correlation of the two images. The cross-correlation is performed in the frequency-domain.

    Cross-correlation (5-steps) - The same algorithm as the normal Cross-correlation, but the shift is only recalculated for every 5th image, therefore reducing the overall calculation time.

    Melt-pool center - The shift is determined by calculating the difference in the position of the center of the melt pool of each image. The melt-pool center is defined as the center of gravity of the melt-pool (not of the complete image). The 'Spatial characteristics' setting has to be enabled for this algorithm.

    The default algorithm is 'Cross-correlation'.

    		Noise treshold

    When image smoothing is enabled, all pixels with an intensity below the noise threshold are changed to black pixels.

    The default value is 25.

    Accepted values are 0-255.

    		Smoothing algorithm

    The algorithm used for the reduction of image noice.

    Values:

    Noise thresholding - This turns all the pixels with an intensity below the specified 'Noise threshold' into black pixels. This filter helps removing background and reduces border artefacts when calculating the temperature field.

    Median filter - This filter replaces each pixel by the median intensity value of the 9 nearest pixels. The filter reduces sensor noise and flattens spikes in the temperature filed.

    Gaussian filter - This filter replaces each pixel by a weighed average (Gaussian weights) of the 9 nearest pixels. The filter has the same effect as the 'Median filter', but is better at preserving the edges of objects in the image.

    Attention: The 'Median filter' and 'Gaussian filter' algorithm also apply the 'Noise thresholding' algorithm. Set the 'Noise threshold' to 255 to avoid this.

    The defautl algorithm is 'Noise thresholding'.

    		2nd image correlation

    Image correlation allows the superposition of two separate images, where there is a small shift in the position of the object on the image.

    Correlation of the 2nd camera image on the 1st camera image is necessary for the calculation of the temperature field.

    		Image smoothing

    Image smoothing reduces the influence of spatial image noise, which can improve the calculation of the temperature field.

    The image smoothing is performed after the melt-pool characteristics calculation, and has thus no effect on its values.

Characteristic params.

    		Binarization treshold

    The minimum pixel intensity level at which a pixel is considered molten.

    This setting impacts the calculation of the melt-pool characteristics.

    The default value is 120.

    Accepted values are 0-255.

    		Image characteristics

    This setting enables the extraction of various image characteristics during the processing of the image. The results can optionally be stored in the 'Characteristics' data collection.

    The extracted characteristics are the average intensity over the complete image and the highest (peak) intensity found.

    Both properties are expressed in 8-bit grey-level values (0-255). The processing is based on the 'Corrected' image, or the original image when no corrections are enabled.

    This setting or the 'Melt-pool characteristics' setting have to be checked in order to be able to store the 'Characteristics' data.

    		Melt-pool characteristics

    This setting enables the extraction of various characteristics of the melt-pool during the processing of the image. The results can optionally be stored in the 'Characteristics' data collection.

    The extracted characteristics are the size of the main melt-pool (in mm²), the ratio of the two principal axes of an ellipse fitted over the melt pool and the total number of melt pools found (detection of melt-pool breakdown or balling).

    This setting or the 'Image characteristics' setting have to be checked in order to be able to store the 'Characteristics' data.

    		Melt-pool spatial charact.

    This setting enables the extraction of information regarding the position and orientation of the melt pool.

    The extracted properties are the position of the center of gravity of the melt pool in pixels and the orientation of the melt pool. The orientation is determined by calculating the principal axis. The orientation is expressed in degrees, starting from the positive x-axis and rotating counter-clockwise.

    This settings has to be checked to be able to save the spatial melt-pool characteristics.

Temperature params.

    		Temperature field

    This setting enables the calculation of the temperature for each pixel in the image.

    The temperature calculation uses the 2-color pyrometer ratio method. Therefore 2 cameras have to be used and 2nd image correlation has to be enabled.

    The correspondence between the ratio of the intensity of the two cameras and the temperature is interpolated from the points in an external file ('temperatureFieldRegression.dat' in 'Program data'). The correspondence is the result of a previous calibration.

    The temperature calculation automically corrects for varying exposure times between the two cameras. An extra correction factor for the intensity ratio (due to filters,...) is available.

    		Ratio correction factor

    The correction factor to be applied to the intensity ratio, before the temperature calculation.

    The default value is 1 (no correction).

    		Temperature value

    This setting enables the calculation of the temperature at a chosen position or region.

    Next to the average temperature, also the minimum and maximum temperature, the standard deviation and the number of pixels taken into account is calculated.

    In order to calculate a temperature value, 2 cameras have to be used and the 'Temperature field' and '2nd image correlation' settings have to be enabled.

    		Y-coord.

    The y-coordinate of the position of the center of the temperature value.

    The y-axis is towards the bottom of the image.

    The unit and axis origin depend on the selection for the 'Averaging center'. If 'Melt-pool center' is selected, this setting is neglected.

    		X-coord.

    The x-coordinate of the position of the center of the temperature value.

    The x-axis is towards the right of the image.

    The unit and axis origin depend on the selection for the 'Averaging center'. If 'Melt-pool center' is selected, this setting is neglected.

    		Averaging center

    Determine the center position for the temperature value calculation.

    Values:

    Fixed coordinates - The center of the temperature value is a fixed point in the image. The coordinates of the point can be given in pixel position (relative to top-left) or in mm (relative to center of image).

    Melt-pool center - The center of the temperature value is re-calculated for each image as the center of gravity of the melt-pool of the 1st camera image. The 'Spatial characteristics' setting has to be enabled for this functionality.

    The default option is 'Fixed coordinates (mm)'.

    		Averaging shape

    Choose the shape over which the temperature value is calculated.

    Values:

    Point - The temperature of a single point at the chosen position is returned.

    Circle - The temperature is averaged over a circular area around the chosen center. The diameter of the circle, in pixels, can be set in 'Major axis'.

    Ellipse - The temperature is averaged over an elliptical area around the chosen center. The major and minor full axis can be set using the correspoding settings. The ellipse major axis is aligned automatically with the melt pool principal axis. For this functionality the 'Spatial characteristics' setting has to be enabled.

    The defualt setting is 'Point'.

    		Minor axis

    The ellipse full minor axis, in pixels.

    If the averaging shape is 'Point' or 'Circle', this setting is neglected.

    		Major axis

    The circle diameter of ellipse full major axis, in pixels.

    If the averaging shape is 'Point', this setting is neglected.
